每个职业一共有1+9个不同巅峰盘，第一个盘start巅峰盘，他有个巅峰等级起始节点，和一个面板链接关口，只有一个而且一点位于巅峰盘上面，所以链接的下一个巅峰盘一点是在start盘的上方。其他9个盘的特征是，一共有四个面板关联接口，而且都位于四边的正中间。如果某一边链接了上一个巅峰盘的出口位置，那这个边的面板管理接口就变成了入口了，负责链接两个盘。第三个盘根据第二个盘里面的路径分别挨着第二个盘展现。以此类推。一共可以链接4个巅峰盘。最终结果是 start盘链接第二个盘，第二盘和三、四盘通过面板链接关口进行链接。另一个重要信息，每个盘在链接时需要支持旋转，分别是0度、90度、180度、270度，这样就可以对接不对的边，内部的点亮节点也可以不一样了。请先了解这些

理解的需求
1. 巅峰盘结构
Start盘：1个起始节点 + 1个Gate（顶部中间）
其他盘：4个Gate（四边正中间）
2. 连接机制
Start盘点击Gate → 弹出选择框选择第2个盘
第2个盘的链接Gate变成入口 → 可以被点亮
点亮节点直到点到这个盘的其他Gate
点击其他Gate → 选择第3个盘
依次往复
3. 数量限制
最多连接4个盘（加上Start盘共5个）
4. 旋转功能
每个盘支持0°、90°、180°、270°旋转
旋转决定Gate的对接位置

架构设计
核心难点是坐标系统：之前每个盘的网格坐标都是"局部坐标"（0~20的行列），现在5个盘要拼在同一张大地图上，需要一套全局坐标系，把每个盘的局部坐标"摆放"到全局画布的某个位置和朝向上。
1. 巅峰盘摆放模型（Board Placement）
每个激活的巅峰盘需要记录：
- boardId: 用的是哪个巅峰盘
- globalOffset: { row, col } —— 这个盘的(0,0)在全局画布上的位置
- rotation: 0° / 90° / 180° / 270° —— 玩家可以旋转拼接方向（游戏里巅峰盘确实能转向拼接）
2. 拼接规则（Snap）
核心盘有1个Gate(顶部)，可选盘有4个Gate(上下左右各一个)。拼接时：新盘的某个Gate必须与已摆放盘的某个Gate的全局坐标重合（贴在一起），这是"拼图"的强制约束，不能随便摆。
3. 跨盘连通性算法
之前 paragonUnlock.ts 里的BFS是"单盘内"的。现在要扩展成"全局坐标"上的BFS——把所有已摆放盘的节点，按全局坐标统一看成一张大图，Gate和Gate重合的位置自动连通（A盘Gate解锁了，等于B盘对应Gate也具备了"被解锁"的前置条件）。
